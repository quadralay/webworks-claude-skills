# Marketing Campaign: WebWorks Claude Skills Plugin Launch

## Overview

Launch marketing campaign to announce the webworks-claude-skills plugin going public, targeting technical writers, documentation professionals, FrameMaker users, and Markdown publishers.

## Target Audiences

| Audience | Primary Pain Points | Key Skills to Highlight |
|----------|-------------------|------------------------|
| **FrameMaker users** | Complex publishing workflows, manual builds | epublisher, automap |
| **Online help creators** | Testing output, theming, CSH validation | reverb2, epublisher |
| **Knowledgebase teams** | Conditional content, variables, automation | markdown-plus-plus, automap |
| **Markdown publishers** | Need professional features without leaving Markdown | markdown-plus-plus |
| **Technical writers** | End-to-end workflow assistance | All four skills |

## Unique Value Proposition

**"AI for Documentation Publishing"** - First Claude Code plugin suite for technical documentation, combining:
- Markdown++ syntax expertise (variables, conditions, includes)
- ePublisher project intelligence
- AutoMap build automation
- Reverb 2.0 output testing

## Messaging Framework

### Elevator Pitch (30 seconds)

> "We built an open-source Claude Code plugin that brings AI assistance to technical documentation. It understands Markdown++, automates ePublisher builds, and tests your output - all through natural language commands."

### Key Messages by Audience

**For FrameMaker users:**
> "Keep using FrameMaker for authoring while AI handles the publishing complexity. AutoMap automates builds, and Claude understands your ePublisher project structure."

**For Markdown authors:**
> "Markdown++ adds variables, conditions, and includes to Markdown - and now you have an AI that's an expert in the syntax. No more looking up documentation."

**For documentation managers:**
> "Reduce publishing errors and testing time. AI catches Markdown++ syntax issues before publishing and tests output in a headless browser."

## Risk Mitigation

### Launch Day Failure Scenarios

| Risk | Mitigation |
|------|-----------|
| Plugin has critical bug | Pre-test with 5+ friendlies, have hotfix process ready |
| HN post gets flagged/ignored | Backup: focus on Reddit + LinkedIn, retry HN next week |
| Landing page down | Use GitHub README as fallback landing page |
| Negative early feedback | Respond quickly, acknowledge issues, fix fast |
| Low engagement | Double down on Write the Docs community (most targeted) |

### Pre-Flight Checklist

Before launch day:
- [ ] 10+ friendlies confirmed to amplify on launch day
- [ ] Plugin tested by non-team-members successfully
- [ ] All social posts drafted and reviewed
- [ ] FAQ responses prepared for common questions
- [ ] Support channel ready (GitHub Issues + email)
- [ ] Team availability confirmed for launch day
